# مرحلة التخطيط  
## نظام إنشاء الجداول الأكاديمية – النسخة المبسّطة (MVP)

---

## 1. تعريف المشروع
**اسم المشروع:** نظام إنشاء الجداول الأكاديمية – كلية الهندسة

**الوصف:**  
تطبيق سطح مكتب مبسّط بلغة **C# (WinForms أو WPF)** يتيح:
- إدخال البيانات الأساسية (المواد، المدرسون النظريون والعمليون، القاعات، الأقسام، المستويات)
- توليد جدول أسبوعي مبسّط
- عرض الجدول بطريقة منظمة
- تصدير / حفظ / طباعة الجدول بصيغة PDF (مع دعم RTL)

---

## 2. الأهداف الرئيسية
1. واجهة بسيطة لإدخال وإدارة البيانات الأكاديمية.  
2. توليد جدول أكاديمي مبسّط يعتمد على القسم والمستوى.  
3. عرض الجدول بطريقة واضحة تتضمن:  
   - الأيام  
   - الوقت  
   - القاعة  
   - الدكتور النظري  
   - الدكتور العملي  
   - القسم  
   - المستوى  
4. إمكان حفظ الجدول وتصديره كملف PDF مع دعم RTL.  
5. إنجاز نسخة MVP بسيطة قابلة للتطوير لاحقًا.

---

## 3. نطاق المشروع (Scope)

### ✔ ضمن النطاق (In Scope)
- واجهة إدخال بسيطة للمكوّنات:
  - مدرس (نظري/عملي)
  - مادة
  - قاعة
  - قسم
  - مستوى  
- عملية توليد جدول مبسّطة دون تعقيدات.  
- لوحة عرض الجدول (أيام × محاضرات) مع جميع الحقول المطلوبة.  
- زر تصدير PDF مع خيارات أساسية.  
- حفظ الجدول في قاعدة بيانات محلية (SQLite أو SQL Server LocalDB).

### ❌ خارج النطاق (Out of Scope)
- خوارزميات جدولة متقدمة.  
- نظام صلاحيات متقدمة.  
- واجهات ويب أو تكامل مع أنظمة جامعية.  
- تحليلات أو إحصاءات.

---

## 4. أصحاب المصلحة (Stakeholders)
- **المهندسة دعاء:** مالكة المحتوى ومدخلة البيانات.  
- **الطالب:** مستفيد نهائي (عرض PDF).  
- **فريق التطوير:** تنفيذ، اختبار، صيانة.

---

## 5. المخرجات المتوقعة (Deliverables)
1. قاعدة بيانات بسيطة (جداول للمكوّنات).  
2. واجهات إدخال: مدرس، مادة، قاعة، قسم، مستوى.  
3. زر توليد الجدول للقسم والمستوى.  
4. لوحة عرض جدول أسبوعي (الأيام × المحاضرات).  
5. ميزة تصدير/حفظ PDF.  
6. مستند مختصر يشرح طريقة التشغيل والتصدير.

---

## 6. المتطلبات العامة (High-Level Requirements)

### 6.1 متطلبات الإدخال
- المدرس النظري  
- المدرس العملي  
- المادة  
- القاعة  
- القسم  
- المستوى الأكاديمي  

### 6.2 متطلبات التوليد
- توليد جدول أسبوعي مبسّط حسب:
  - القسم
  - المستوى

### 6.3 متطلبات العرض
الجدول يعرض الحقول:
- اليوم  
- الوقت  
- المادة  
- القاعة  
- الدكتور النظري  
- الدكتور العملي  
- القسم  
- المستوى  

### 6.4 متطلبات PDF
- استخراج الجدول كملف PDF  
- دعم RTL  
- تنسيق بسيط جاهز للطباعة  

### 6.5 متطلبات التشغيل
- نظام Windows  
- C# (WPF أو WinForms)  
- قاعدة SQLite أو LocalDB  

### 6.6 متطلبات الواجهة
- واجهة بسيطة  
- خالية من التعقيد  
- مناسبة لمستخدم غير تقني  
- عرض واضح للمدخلات والجدول

---

## 7. الخاتمة
تم في هذه المرحلة تحديد:
- فكرة المشروع  
- الأهداف  
- نطاق العمل  
- أصحاب المصلحة  
- المخرجات  
- المتطلبات العامة  

وتقدّم الوثيقة رؤية واضحة للنسخة الأولى من النظام (MVP)، مما يمهّد للانتقال السلس لمرحلة **التحليل** ثم **التصميم**.

تمثل هذه المرحلة الأساس المتين لبقية مراحل بناء النظام وتضمن وضوح المسار وتجنّب تضخم المتطلبات.
